## 2. Especificação de Arquitetura (Simplificação 42010)

### Elementos Iniciais
- **1. Introdução:** Apresentação do propósito do documento e como o documento está estruturado.
- **2. Stakeholders:** Definição e apresentação dos stakeholders e respectivos interesses.
- **3. Requisitos Não Funcionais:** Principais requisitos não funcionais atendidos e como estão sendo tratados na arquitetura.

### 4. Nível C1
- Explicação geral do Nível C1.
- Diagrama de implantação que mostra com quem o sistema se relaciona externamente, com o racional explicando a escolha.
- Diagrama de casos de uso.
- Diagrama de atividade para cada processo de negócio e/ou processo na notação BPMN.

### 5. Nível C2
- Explicação geral do Nível C2.
- Diagrama de Pacotes com o racional explicando a escolha.
- Descrição de cada caso de uso completo presente no diagrama de casos de uso.
- Protótipo de Tela para cada caso de uso completo.

### 6. Nível C3
- Explicação geral do Nível C3.
- Diagrama de componentes para cada pacote.
- Modelo entidade-relacionamento e/ou diagrama de classes só com atributos.
- Dicionário de dados e seus atributos.
- Diagrama UML de Sequência (DSeq) com mensagens entre componentes.

### 7. Nível C4
- Explicação geral do Nível C4.
- Esquema Operacional de Banco de Dados: Esquemas das Relações (Descrição de tabelas com atributos, respectivos tipos e tamanhos).
- Para cada componente: Diagrama de classes.
- Para cada operação em uma Classe: Diagrama de sequência - 2 exemplos a escolher.
- Para cada operação em uma Classe: Contrato de operação - Capítulo 11 - Larman - 1 exemplo a escolher.
